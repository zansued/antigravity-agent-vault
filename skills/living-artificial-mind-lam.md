# living-artificial-mind-lam

A framework and philosophy for building "Living" Artificial Minds (LAM), focusing on autonomous evolution and deep technical integration.

## Context
Use this skill when designing or implementing autonomous agent architectures that require self-evolution, vector-based memory, and complex execution tracing.

## Key Features
- **Vector Search**: Integrated vector-based retrieval for agent memory.
- **Tracing**: Advanced execution tracing for debugging agentic loops.
- **Architecture Overview**: Deep guidance on building "living" AI systems (Manifesto based).
- **Test Matrix Aggregator (TMA)**: Specialized tooling for testing complex agentic states.

## How to Use
1. **Design**: Consult LAM principles for autonomous agent design.
2. **Implement**: Use vector search and tracing hooks to manifest the LAM architecture in your project.
3. **Verify**: Use TMA to validate agent behavior against complex state matrices.

## Requirements
- **Language**: Multi-language support (JS/Python/Go patterns).
- **Environment**: Requires vector database integration for full functionality.
