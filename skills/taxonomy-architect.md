# taxonomy-architect

A specialized tool for architecting and managing complex taxonomies using "The Razor" philosophy for structured data organization.

## Context
Use this skill when you need to design, refactor, or manage hierarchical data structures, category systems, or knowledge graphs with high precision.

## Key Features
- **The Razor Philosophy**: Enforces strict, non-redundant, and logical taxonomy structures.
- **Architectural Pipeline**: A structured workflow for taxonomy creation, from discovery to implementation.
- **Hierarchical Management**: Tools for visualizing and modifying deep category trees.
- **Structured Output**: Generates implementation-ready schemas for various databases and knowledge bases.

## How to Use
1. **Architect**: "Design a taxonomy for [domain] following The Razor philosophy".
2. **Refactor**: "Analyze the current category structure and remove redundancies using the architect".
3. **Pipeline**: "Start the taxonomy pipeline for the new product catalog".

## Requirements
- **Runtime**: CLI or Node.js.
- **Philosophy**: Requires adherence to "The Razor" principles for best results.
