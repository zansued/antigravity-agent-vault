# critical-thinking-skepticism

A specialized command extension (`/criticalthink`) that embeds healthy skepticism into the AI reasoning process to counter confirmation bias and authority bias.

## Context
Use this skill when dealing with high-stakes decisions, complex architectural reviews, or whenever you need a "second opinion" from a skeptical perspective.

## Key Features
- **Confirmation Bias Mitigation**: Specifically designed to challenge the AI's tendency to agree with its own initial conclusions.
- **Mutual Criticism Process**: Implements a framework where the agent actively looks for failure patterns in its proposed solutions.
- **Skepticism Framework**: Uses a structured approach to analyze potential logic flaws, edge cases, and architectural weaknesses.
- **Bias Countermeasures**: Helps the user avoid the "authority bias" of blindly trusting AI-generated responses.

## How to Use
1. **Analyze**: "/criticalthink Review the proposed [AlgorithmChange] and identify potential logic flaws".
2. **Challenge**: "Apply a skeptical analysis to the [SecurityPlan] to find hidden vulnerabilities".
3. **Refine**: "Use the critical thinking framework to improve the robustness of the [DesignChoice]".

## Requirements
- **Runtime**: Gemini CLI / Claude Code.
- **Integration**: Works as a top-level slash command extension.
