# conductor-beads-persistent-memory

A robust framework for persistent memory, session resumption, and parallel task execution for AI coding agents.

## Context
Use this skill to give your AI agent long-term memory, allow it to resume complex multi-step sessions, and coordinate parallel task execution.

## Key Features
- **Persistent Memory**: Uses "Beads" to store session context and agent state across restarts.
- **Knowledge Flywheel**: Implements "Ralph-style" learnings to continuously improve agent performance based on past interactions.
- **Parallel Execution**: Coordination layer for splitting complex tasks into parallel, executable tracks.
- **Session Resumption**: Seamlessly resume interrupted workflows with full state restoration.
- **Cross-Platform**: Optimized for Gemini CLI and Claude Code.

## How to Use
1. **Initialize**: "Initialize Conductor-Beads for the current project to enable persistent memory".
2. **Execute**: "Split the [TaskName] into parallel tracks and execute using Beads".
3. **Resume**: "Resume the previous session and apply knowledge from the latest learnings".

## Requirements
- **Tools**: Beads runtime installed.
- **Framework**: Gemini CLI or Claude Code integration enabled.
