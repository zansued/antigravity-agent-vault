# outline-driven-development-core

The core repository for the Outline Driven Development (ODD) paradigm, providing AST analysis and modern context engineering for augmented LLM code agent workflows.

## Context
Use this skill to implement the ODD paradigm at a structural level, leveraging AST analysis and native-optimized CLI tools to transcend basic "vibes" or overly complex "specs".

## Key Features
- **AST Analysis**: Uses Abstract Syntax Tree analysis for deterministic code understanding.
- **Context Engineering**: Advanced techniques for injecting precise technical context into LLM prompts.
- **Rust-Based CLI Tools**: Includes high-performance, native-optimized tools (through cargo) for development workflows.
- **Batteries-Included**: A complete set of best practices and tools for deterministic-with-non-deterministic-LLM behavior.
- **Paradigm Shift**: Moves beyond traditional documentation into executable outlines.

## How to Use
1. **Analyze**: "Run ODD AST analysis on the current module to identify structural patterns".
2. **Engineer**: "Apply context engineering to optimize the prompt for the [TaskName] phase".
3. **Execute**: "Follow the ODD batteries-included workflow for the next implementation cycle".

## Requirements
- **Tools**: Requires Rust/Cargo for native CLI tool optimizations.
- **Methodology**: Strict adherence to the ODD paradigm for optimal results.
