---
description: How to ship a SaaS MVP from scratch using agentic skills
---

1. Define MVP boundaries and acceptance criteria.
   - Use `@concise-planning` or `@brainstorming`
2. Implement core entities, APIs, and auth baseline.
   - Use `@backend-dev-guidelines` and `@api-patterns`
3. Ship core user flow with clear UX states.
   - Use `@frontend-developer` and `@shadcn-ui-design-system`
4. Cover critical user journeys before release.
   - Use `@browser-automation` or `@test-driven-development`
5. Release with observability and rollback plan.
   - Use `@deployment-procedures` and `@observability-engineer`
