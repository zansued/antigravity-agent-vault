---
description: Comprehensive security audit workflow for web applications
---

1. Identify assets, trust boundaries, and attack paths.
   - Use `@threat-modeling-expert` or `@ethical-hacking-methodology`
2. Detect account takeover and authorization flaws.
   - Use `@auth-implementation-patterns` and `@broken-authentication`
3. Uncover high-impact API and injection vulnerabilities.
   - Use `@api-security-best-practices` and `@top-web-vulnerabilities`
4. Convert findings into fixes and verify evidence.
   - Use `@security-auditor` and `@verification-before-completion`
5. Finalize security posture.
   - Use `@behavioral-security-threat-orchestration`
