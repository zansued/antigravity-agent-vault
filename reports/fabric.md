# Relatório de Absorção: Fabric (Daniel Miessler)

**Data:** 13 de Março de 2026
**Analista:** Gemini CLI (Agentic Mode)

## 1. Filosofia de "Prompt Engineering"
O Fabric não trata prompts como simples perguntas, mas como **Programas de Pensamento**:
- **Identity & Context:** Define IAs com QIs altíssimos (ex: 1419) e especialidades ultra-específicas para mudar o "baseline" do modelo.
- **Raciocínio Espacial e Temporal:** Usa instruções hiperbólicas ("Whiteboard de 100m", "Pense por 312 horas", "Re-leia 218 vezes") para forçar a expansão do contexto latente do modelo.
- **Iteração Recursiva:** Comanda o modelo a realizar ciclos de otimização internos (ex: "Repita este passo 913 vezes") antes de gerar a saída.

## 2. Estrutura de Saída (Output Rigor)
- **Constraints de Tamanho:** Uso frequente de "15-word bullets" para garantir densidade de informação e evitar verbosidade.
- **Seções Semânticas:** Divisão clara em seções como `IDENTITY`, `GOALS`, `STEPS`, `OUTPUT` e `POSITIVE/NEGATIVE EXAMPLES`.
- **Markdown Limpo:** Preferência por Markdown sem negritos/itálicos para máxima legibilidade em terminais e sistemas de texto puro.

## 3. Categorias de Patterns
- **Extraction:** Wisdom, Insights, Quotes, Algorithms.
- **Analysis:** Code Review, Threat Modeling, Personality, Risks.
- **Creation:** Design Docs, PRDs, Coding Projects, Academic Papers.
- **Summarization:** Meeting Minutes, Video Chapters, Micro-summaries.

## 4. Integração para Gemini CLI
- Adotar o padrão de **"Identity Framing"** para meus sub-agentes (ex: "Você é um Engenheiro de Segurança com 20 anos de experiência em Kernel").
- Implementar **"Thought Expansion Steps"** em tarefas complexas de análise arquitetural.
- Utilizar a técnica de **"15-word bullet constraints"** para gerar resumos executivos de alta fidelidade para o usuário.
